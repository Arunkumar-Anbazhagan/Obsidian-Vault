## ðŸ“Œ Introduction to Nodes
* **[00:00:08](https://www.youtube.com/watch?v=rCPXBkeBWCQ&t=8)**: A **Node** is the fundamental building block (the "atom") of all n 8 n workflows.
* **[00:00:26](https://www.youtube.com/watch?v=rCPXBkeBWCQ&t=26)**: Every workflow is built by connecting consecutive nodes.

## ðŸ—‚ï¸ Categories of Nodes
* **[00:00:36](https://www.youtube.com/watch?v=rCPXBkeBWCQ&t=36)**: There are three main categories:
    1.  **Entry Points (Triggers)**: What starts the workflow.
    2.  **Functions**: Transform, filter, or format data.
    3.  **Exit Points (Actions/Apps)**: Interaction with external applications (e.g., Google Sheets, Slack).

## ðŸ—ï¸ Building a Workflow (The Canvas)
* **[00:01:14](https://www.youtube.com/watch?v=rCPXBkeBWCQ&t=74)**: Workflows are built on the **Canvas**.
* **[00:01:21](https://www.youtube.com/watch?v=rCPXBkeBWCQ&t=81)**: **Adding the First Node**:
    * Click "Add first step" or the `+` button.
    * The first node is usually a **Trigger** (e.g., Manual Trigger, On App Event).
* **[00:02:07](https://www.youtube.com/watch?v=rCPXBkeBWCQ&t=127)**: **Adding Subsequent Nodes**:
    * Search for the app or function by name.
    * **[00:02:24](https://www.youtube.com/watch?v=rCPXBkeBWCQ&t=144)**: Select an **Operation** (Action). For example, in Google Sheets: `Append`, `Update Row`, `Clear Sheet`, etc.

## âš™ï¸ Node Configuration
* **[00:03:13](https://www.youtube.com/watch?v=rCPXBkeBWCQ&t=193)**: Double-clicking a node opens its settings.
    * **Parameters**: Specific to the node and operation (e.g., selecting the Spreadsheet ID).
    * **[00:04:02](https://www.youtube.com/watch?v=rCPXBkeBWCQ&t=242)**: **Credentials**:
        * Essential for authenticating with external services (APIs).
        * Saved at the instance level (can be reused across workflows).
        * Can support OAuth (e.g., "Sign in with Google").

## ðŸ“Š Input & Output Data Views
* **[00:04:41](https://www.youtube.com/watch?v=rCPXBkeBWCQ&t=281)**: To the left of parameters is **Input Data**, to the right is **Output Data**.
* **[00:05:00](https://www.youtube.com/watch?v=rCPXBkeBWCQ&t=300)**: Three main data views:
    1.  **Table View**: Columns and values (like a spreadsheet).
    2.  **JSON View**: Key-value pairs (covered in the next video).
    3.  **Schema View**: Keys from the input and example values.

## ðŸ› ï¸ Practical Demo: Google Sheets
* **[00:05:56](https://www.youtube.com/watch?v=rCPXBkeBWCQ&t=356)**: **Goal**: Read data from a Google Sheet.
    1.  **Trigger**: Manual Trigger.
    2.  **Action Node**: Google Sheets -> "Get Many Rows".
    3.  **Credential Setup**: Using OAuth to sign in with Google.
    4.  **Configuration**: Select the Document and specific Sheet (e.g., "Data").
    5.  **Execution**: Clicking "Execute Node" runs it and shows the data in the Output panel.

## ðŸ”§ Advanced Node Settings
* **[00:10:52](https://www.youtube.com/watch?v=rCPXBkeBWCQ&t=652)**: Accessible via the gear icon or "Settings" tab. These are independent of the specific node type.
    * **Execute Once**: Toggle to run only once or for every item.
    * **Retry on Fail**: Useful for unstable APIs.
    * **Error Handling**: Define what happens if the node fails (Stop workflow vs. Continue).
    * **Notes**: Add comments to explain what the node does for better readability.
