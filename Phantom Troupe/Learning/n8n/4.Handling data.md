# üì¶ How does n 8 n handle data? (n 8 n Beginner Course)

**Instructor:** Maxim
**Video:** [n8n Beginner Course (4/9) - How does n8n handle data?](https://youtu.be/2YfWuNziPE4?si=pepA2gG6fj-YJ_XW)

## üìå Overview
* **[00:00:01](https://youtu.be/2YfWuNziPE4?t=1)**: This video covers core technical concepts: what kind of data nodes use and how data flows from node to node. Understanding this is crucial for building effective workflows.

## üß± Core Data Structures
* **[00:00:41](https://youtu.be/2YfWuNziPE4?t=41)**: Two main data structures in n 8 n:
    1.  **JSON (JavaScript Object Notation)**:
        * Written between curly braces `{}`.
        * Made up of **Key-Value pairs** separated by commas.
        * **[00:01:31](https://youtu.be/2YfWuNziPE4?t=91)**: Can be **embedded** (nested), meaning a value can be another JSON object (e.g., grouping location data like `{ "location": { "city": "Paris" } }`).
        * **[00:02:11](https://youtu.be/2YfWuNziPE4?t=131)**: Accessed via **Dot Notation** (e.g., `json.first_name` or `json.location.country`).
    2.  **Lists (Arrays)**:
        * A collection of objects written between square brackets `[]`.
        * Can contain a mix of types (letters, numbers) or objects.
        * **[00:03:29](https://youtu.be/2YfWuNziPE4?t=209)**: In n 8 n, you typically work with a **List of JSONs**.

## üîÑ The n 8 n Data Format: "Items"
* **[00:04:01](https://youtu.be/2YfWuNziPE4?t=241)**: **JSON vs. Tables**:
    * One JSON object is equivalent to one **Row** in a table.
    * Keys are the **Headers** (Columns).
* **[00:05:08](https://youtu.be/2YfWuNziPE4?t=308)**: **Items**:
    * n 8 n calls each JSON object in the list an **Item**.
    * Nodes accept **Items** (plural) as input and return **Items** as output.
    * Even if there is no data, a node returns a list with an empty JSON `{}`.

## ‚öôÔ∏è How Nodes Execute
* **[00:05:49](https://youtu.be/2YfWuNziPE4?t=349)**: **Execution Logic**:
    * **Default Rule**: Each node executes **once per item** in the input data.
    * *Example*: If you input 3 dates into a formatting node, it executes 3 times and outputs 3 formatted dates.
* **[00:07:10](https://youtu.be/2YfWuNziPE4?t=430)**: **Exceptions**:
    * You can change settings (gear icon) to **"Execute Once"**, which forces the node to run only for the first item.

### üåä Execution Flow Example ([00:07:27](https://youtu.be/2YfWuNziPE4?t=447))
1.  **Start**: Returns 1 empty item `[{}]`.
2.  **Google Sheets Node**: Reads the sheet and returns 3 items (rows) `[{}, {}, {}]`.
3.  **Filter Node**: Checks each of the 3 items against a condition. If only 1 matches, it outputs 1 item `{}`.

## üß™ Expressions
* **[00:09:02](https://youtu.be/2YfWuNziPE4?t=542)**: Used to configure nodes dynamically based on input data.
* **How to create**: Drag and drop a key from the Input Data panel into a parameter field.
* **[00:10:38](https://youtu.be/2YfWuNziPE4?t=638)**: **Syntax**:
    * Written between double curly braces `{{ }}`.
    * Can reference item variables: `{{ $json.first_name }}`.
    * Can use **JavaScript**: `{{ $json.last_name.toUpperCase() }}`.
    * Can combine text and variables: `{{ $json.first_name }} signed up`.

## üõ†Ô∏è Practical Demo: Using Expressions
* **[00:12:33](https://youtu.be/2YfWuNziPE4?t=753)**: **Goal**: Create a "Full Name" field.
    1.  Use the **Edit Fields** node (or "Set" node).
    2.  Create a new field named `full_name`.
    3.  Drag `first_name` and `last_name` from input to value.
    4.  Resulting Expression: `{{ $json.first_name }} {{ $json.last_name }}`.
    5.  **[00:14:08](https://youtu.be/2YfWuNziPE4?t=848)**: Added JavaScript `.toUpperCase()` to capitalize the last name directly inside the expression.
